Data Locker Cohort

고급 코호트 분석을 통한 사용자 행동 인사이트

Data Locker Cohort란?

Data Locker Cohort는 AppsFlyer의 강력한 코호트 분석 도구로, 사용자 그룹의 행동을 시간에 따라 추적하고 분석할 수 있습니다. 이를 통해 마케팅 캠페인의 효과, 사용자 유지율, 수익성 등을 심층적으로 분석할 수 있습니다.

기존 코호트 분석과의 차이점

Data Locker Cohort는 기존의 대시보드나 API 기반 코호트 분석과 비교하여 다음과 같은 장점을 제공합니다:

Data Locker Cohort의 장점

  • 더 많은 데이터: 대시보드에서 제공하는 것보다 더 많은 원시 데이터에 접근할 수 있습니다.
  • 유연한 분석: SQL 쿼리를 통해 원하는 방식으로 데이터를 분석할 수 있습니다.
  • 사용자 정의 지표: 비즈니스에 맞는 맞춤형 지표를 생성할 수 있습니다.
  • 긴 기간 분석: 더 긴 기간의 데이터를 분석할 수 있습니다.
  • 데이터 통합: 다른 데이터 소스와 쉽게 통합할 수 있습니다.
  • 자동화: 분석 프로세스를 자동화하여 정기적인 보고서를 생성할 수 있습니다.

주요 사용 사례

LTV 분석

사용자 획득 채널별 고객 생애 가치(LTV)를 분석하여 가장 수익성 높은 마케팅 채널을 식별합니다.

유지율 분석

다양한 사용자 세그먼트의 유지율을 비교하여 이탈 원인을 파악하고 개선 전략을 수립합니다.

캠페인 성과 분석

마케팅 캠페인의 장기적인 성과를 측정하여 ROI를 최적화합니다.

사용 사례: 미디어 소스별 LTV 분석

다음은 Data Locker Cohort를 사용하여 미디어 소스별 사용자 LTV를 분석하는 예제입니다.

샘플 데이터

아래는 Data Locker에서 제공하는 샘플 데이터 구조입니다:

appsflyer_id install_date media_source campaign event_date event_name revenue
id123 2025-01-01 Facebook Spring_Sale 2025-01-05 purchase 15.99
id456 2025-01-02 Google Winter_Promo 2025-01-10 purchase 24.99
id789 2025-01-03 TikTok New_User 2025-01-15 purchase 9.99

SQL 쿼리 예제

-- 미디어 소스별 30일 LTV 분석

SELECT

media_source,

COUNT(DISTINCT appsflyer_id) AS users,

SUM(CASE WHEN DATEDIFF(event_date, install_date) <= 30 THEN revenue ELSE 0 END) AS revenue_30d,

SUM(CASE WHEN DATEDIFF(event_date, install_date) <= 30 THEN revenue ELSE 0 END) / COUNT(DISTINCT appsflyer_id) AS ltv_30d

FROM

app_events

WHERE

install_date BETWEEN '2025-01-01' AND '2025-01-31'

AND event_name = 'purchase'

GROUP BY

media_source

ORDER BY

ltv_30d DESC;

분석 결과

위 분석 결과를 통해 Google 광고가 가장 높은 30일 LTV를 보이는 것을 확인할 수 있습니다. 이러한 인사이트를 바탕으로 마케팅 예산을 더 효율적으로 배분할 수 있습니다.

추가 사용 사례

리인게이지먼트 분석

재참여 캠페인의 효과를 분석하여 이탈 사용자를 다시 활성화하는 최적의 전략을 찾습니다.

사용자 세그먼트 분석

다양한 사용자 세그먼트의 행동 패턴을 분석하여 타겟팅 전략을 개선합니다.

이벤트 기반 분석

특정 이벤트(예: 첫 구매)를 기준으로 코호트를 생성하여 사용자 여정을 분석합니다.

시작하기

Data Locker Cohort를 시작하려면 다음 단계를 따르세요:

  1. AppsFlyer 계정에서 Data Locker를 활성화합니다.
  2. 클라우드 스토리지(AWS S3, Google Cloud Storage 등)를 연결합니다.
  3. 데이터 전송 일정을 설정합니다.
  4. SQL 쿼리 도구(예: Amazon Athena, Google BigQuery)를 사용하여 데이터를 분석합니다.
  5. 분석 결과를 시각화하여 인사이트를 도출합니다.

자세한 내용은 Data Locker Cohort 공식 문서를 참조하세요.
