AppsFlyer 데이터 전송 방식 개요

AppsFlyer는 마케팅 데이터를 다양한 방식으로 제공합니다. 각 방식은 서로 다른 사용 사례와 요구 사항에 맞게 설계되었습니다.

Pull API

필요할 때 데이터를 요청하여 가져오는 방식

Push API

이벤트 발생 시 실시간으로 데이터를 전송받는 방식

Data Locker

대용량 데이터를 안정적으로 전송하는 고급 솔루션

Pull API

일상 비유: 전화를 걸어 음식을 배달 주문하는 것과 같습니다. 필요할 때 요청하고 데이터를 받아옵니다.

Pull API의 특징

  • 요청 기반: 데이터가 필요할 때 API를 호출하여 가져옵니다.
  • 유연성: 필요한 데이터만 선택적으로 가져올 수 있습니다.
  • 배치 처리: 일정 기간의 데이터를 한 번에 가져올 수 있습니다.
  • 적합한 사용 사례: 정기적인 보고서, 대시보드 업데이트, 비정기적 분석

// Pull API 예제 요청

GET https://hq.appsflyer.com/export/app_id/installs_report/v5

?api_token=YOUR_API_TOKEN

&from=2025-01-01

&to=2025-01-31

&additional_fields=install_app_store,match_type,contributor1_match_type

자세한 내용은 Pull API 공식 문서를 참조하세요.

Push API

일상 비유: 은행에서 입출금이 발생할 때마다 알림을 받는 것과 같습니다. 이벤트가 발생할 때마다 실시간으로 데이터를 전송받습니다.

Push API의 특징

  • 실시간 전송: 이벤트 발생 시 즉시 데이터를 전송합니다.
  • 웹훅 기반: 지정된 엔드포인트로 데이터를 자동 전송합니다.
  • 낮은 지연 시간: 데이터를 거의 실시간으로 받을 수 있습니다.
  • 적합한 사용 사례: 실시간 대시보드, 즉각적인 마케팅 대응, 사기 탐지

// Push API 웹훅 수신 예제 (Node.js)

app.post('/appsflyer-webhook', (req, res) => {

const eventData = req.body;

console.log('Received AppsFlyer event:', eventData);

// 이벤트 처리 로직

if (eventData.event_name === 'install') {

// 설치 이벤트 처리

}

res.status(200).send('Event received');

});

자세한 내용은 Push API 공식 문서를 참조하세요.

Data Locker

일상 비유: 송유관과 같습니다. 대량의 데이터를 안정적이고 지속적으로 공급하는 튼튼한 시스템입니다.

Data Locker의 특징

  • 대용량 데이터: 대규모 데이터셋을 효율적으로 처리합니다.
  • 클라우드 통합: AWS S3, Google Cloud Storage, Azure Blob Storage와 같은 클라우드 스토리지와 직접 통합됩니다.
  • 자동 전송: 설정된 일정에 따라 데이터를 자동으로 전송합니다.
  • 데이터 보존: 장기간 데이터 보존 및 접근이 가능합니다.
  • 적합한 사용 사례: 빅데이터 분석, 머신러닝, 고급 비즈니스 인텔리전스

Data Locker는 AppsFlyer의 가장 강력한 데이터 전송 솔루션으로, 엔터프라이즈급 데이터 분석을 위한 최적의 선택입니다.
